# Few-shot Examples : 몇 개의 예시 제공하는 프롬프팅 기법
다양한 레이블 분포를 제공할 때 결과가 잘 나온다.

# CoT(Chain-of-Thought) : 단계별 해결 예시 제공하는 프롬프팅 기법
문제를 풀 때 단계별 해결 예시를 제공하면 문제를 더 잘 푼다.

# Zero-shot CoT : “단계별로 생각해봅시다”라고 하는 프롬프팅 기법
예시를 주지 않고도, 단순히 '단계별로 생각해봅시다'라고 지시만 해도 문제를 더 잘푼다.

# Self-Evaluation : 답변 자체 평가하는 프롬프팅 기법
오답을 낸 경우, 오답을 자체 평가해서 틀린 이유를 설명하라고 하면 문제를 다시 잘 푼다.

# Temperature : 창의성 정도 조절하는 프롬프트 파라미터
temperature가 클수록 창의적인 답변을 하게 된다.

# Presence penalty : 응답에 새로운 단어가 포함될 확률을 조정하는 프롬프트 파라미터
값 범위: -2.0에서 2.0까지의 실수. 
값이 높을수록 (e.g., 1.0): 새로운 단어가 포함될 확률이 높아져 더 창의적인 응답이 생성됩니다.

# Top-p : 이 파라미터는 확률 질량의 누적 합계를 기준으로 토큰을 샘플링하는 프롬프트 파라미터
값이 낮을수록 (e.g., 0.1) 확률이 높은 토큰들만 선택되어 매우 보수적이고 결정적인 응답을 생성합니다.

# Frequency penalty : 특정 단어가 반복되는 것을 얼마나 페널티를 줄 것인지 조정하는 프롬프트 파라미터
값 범위: -2.0에서 2.0까지의 실수. 값이 높을수록 (e.g., 1.0): 동일한 단어가 반복될 확률이 낮아져 응답이 더 다양해집니다.

# Max tokens : 생성할 응답의 최대 토큰 수를 설정하는 프롬프트 파라미터
응답 길이를 제한하여 너무 길거나 짧지 않게 조절할 수 있습니다.
예시: Max tokens 값을 50으로 설정하면, 모델은 최대 50개의 토큰으로 응답을 생성합니다.
